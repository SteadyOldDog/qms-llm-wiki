# Wiki 模式

## 领域
质量管理体系（QMS）与生物安全管理体系——涵盖实验室质量管理、生物安全管理、认可认证、标准化体系、风险管理、人员资质管理等。

适用场景：疾控中心实验室、医学检验实验室、病原微生物实验室、生物安全实验室的质量与安全管理体系建设和运行。

## 约定
- 文件名：小写，连字符分隔，无空格（如 iso-15189.md）
- 每个wiki页面以YAML前置元数据开头
- 使用 [[wikilinks]] 在页面之间链接（每个页面至少2个出站链接）
- 更新页面时，始终更新 updated 日期
- 每个新页面必须添加到 index.md 的正确分类下
- 每个操作必须追加到 log.md
- **溯源标记：** 在综合3个以上来源的页面上，段落末尾附加 ^[raw/articles/source-file.md]

## 前置元数据
```yaml
---
title: 页面标题
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: entity | concept | comparison | query
tags: [来自下方分类法]
sources: [raw/articles/source-name.md]
confidence: high | medium | low
contested: true
contradictions: [其他页面slug]
---
```

## 页面阈值
- **创建页面：** 当实体/概念在2个以上来源中出现，或是一个来源的核心内容
- **追加到现有页面：** 当来源提及已覆盖的内容时
- **不创建页面：** 仅被提及、次要细节或超出领域范围的内容
- **拆分页面：** 当页面超过200行时，拆分为子主题并交叉链接
- **归档页面：** 当内容已被完全替代时，移至 _archive/

## 实体页面
每个实体一页。包含：概述/定义、关键事实与日期、与其他实体的关系、来源引用

## 概念页面
每个概念/主题一页。包含：定义/解释、当前知识状态、开放问题或争议、相关概念

## 比较页面
并列分析。包含：比较对象与原因、比较维度（首选表格格式）、结论或综合、来源

## 更新策略
当新信息与现有内容冲突时：
1. 检查日期——较新的来源通常优先
2. 若确实矛盾，同时标注两种观点、日期和来源
3. 在前置元数据中标注矛盾：contradictions: [页面名]
4. 在检查报告中标记供用户审核

## 来源优先级
1. 法律法规和国家标准（GB、WS、SN等）
2. 国际标准（ISO、IEC）
3. 行业主管部门发布的技术规范
4. 同行评审期刊文章
5. 认可机构发布的指南和要求
6. 权威教材和专业书籍
